# أنماط البوستات | ماركيتيا (SOP)

**Category:** SOP / Content
**Related Topics:** writing-style, terminology, writer-agent, content-planning

> الأمثلة الحقيقية تحت من القناة — مرجع لأسلوب خالد الفعلي.

---

## ١. بوست التحديث السريع (3-6 أسطر)

**الهيكل:**
```
🔸 [الوضع الحالي بجملة]
🔸 [التأثير على السوق]
🔸 [الخطوة المطلوبة من المتابع]
```

**مثال حقيقي:**
```
🔸 تحديث
ارتداد usdt دومينانس من مستوي 4.7% بثبات الارتداد يعطي متنفس للعملات
🔸 برجاء استغلال الصعود في تعديل وضع المحفظة وتوفير بعض الكاش
```

---

## ٢. بوست التحليل (6-12 سطر)

**الهيكل:**
```
🔸 [السؤال اللي في بال المتابع / الوضع الحالي]
🔸 [التحليل والسيناريو الأول]
🔸 [السيناريو الثاني أو الشرط]
🔸 [جملة تلخيصية تربط السيناريوهات ببعض — اختياري]
🔸 [الخلاصة والتوصية]
```

**ملاحظة أسلوبية:** خالد أحياناً بيضيف **جملة تلخيصية في النص** بتربط السيناريوهات قبل الخلاصة — مثال: "دي كلها محطات هنشوفها للسعر في الفترة الجاية حال عدم الثبات أعلى مناطق 65K". ده جزء من أسلوبه مش حشو.

**مثال حقيقي:**
```
🔸 حتي الآن يعجز الكوين عن اغلاق اعلي مقاومه 65500
🔸 حتي الان usdt دوميناس منطلق من ملامسته لمنطقة طلب
🔸 هذا لا يفيد العملات
```

> 🔎 ربط مهم: رغم إن المثال بيوضح أكتر من مستوى، قاعدة الـ Technical Agent إن المخرج **سيناريو واحد أقوى بنقطة إبطال** (مش سيناريوهين متساويين). الجملة التلخيصية بتربط المحطات، مش بتطرح مسارين متعادلين. (راجع `05_agents/technical-agent.md`).

## ٣. بوست تعليمي (8-15 سطر)

**الهيكل:**
```
[عنوان الفكرة / السؤال]

[الشرح على مراحل — مرقمة أو بنقاط]

[الخلاصة والدرس]
```

**مثال حقيقي:**
```
في شغلة مهمة لازم يعرفها المبتدئين...
السوق بمر ب ٣ مراحل:
١- التجميع (قبل الصعود)
٢- مرحلة المشاركة (وقت الصعود)
٣- التصريف (بعد الصعود)
```

---

## ٤. بوست Q&A (أسئلة وأجوبة)

**الهيكل:**
```
- [سؤال المتابع]
الجواب [إجابة مباشرة وواضحة]

- [سؤال ثاني]
الجواب [إجابة]
```

---

## ملاحظات عامة

- العناوين بسيطة: "تحديث لوضع الكوين" — مفيش framing مسرحي.
- البوست التعليمي بيبدأ بسؤال أو فكرة ثم الشرح ثم الخلاصة.
- البوست التحليلي بيبدأ بالوضع الحالي ثم السيناريوهات ثم الخلاصة.
