# تقييم الأداء | ماركيتيا (SOP)

**Category:** SOP / Evaluation
**Related Topics:** performance-tracker-agent, technical-agent, operational-tables

> المرجع الرسمي لقواعد محاسبة التوقعات. Performance Tracker Agent بيطبّقه حرفياً. كل القواعد دي مستقرة من جلسة التقييم الأولى الحية + الـ Master Plan.

---

## نظام الأحكام الأربعة

| الحكم | الرمز | المعنى |
|-------|-------|--------|
| Full Success | 🟢 | السيناريو اتحقق |
| Partial | 🟡 | جزء اتحقق وجزء لأ |
| Neutral | ⚪ | مش ممكن الحكم (شرطي / وقت كافي / مستوى ما وصلش) |
| Failed | 🔴 | خالف التوقع |

لكل توقع: **Impact Score (1-10)** + **Confidence Level (Low / Medium / High)**.

---

## قواعد التقييم (لا تخرج عنها أبداً)

1. **الوصول = نجاح:** السعر وصل المستوى المتوقع = **Full Success 🟢** — حتى لو ما انعكسش بعد.
2. **الشرطي = مفتوح:** المستويات الشرطية (مثل "لو كسر 53K" أو 42K) = **Neutral ⚪** — عدم وصولها **مش فشل**.
3. **التقييم holistic:** قيّم السيناريو كله (اتجاه + هدف تصحيح) ككل، مش الأرقام منفردة. انحراف بسيط في رقم واحد **ما بيحوّلش** Full Success لـ Partial طالما الصورة الكبيرة صح.
4. **القاعدة الذهبية:** ممنوع تغيير معايير التقييم بعد النشر إطلاقاً.
5. **السياق أولاً:** اطلب السياق الكامل للتحليل الأصلي قبل أي حكم.
6. **المقارنة ذاتية:** قارن كل Agent بنفسه عبر الزمن — مش cross-type ranking (مش بنقارن Technical بـ Market).
7. **احترام التوقيت:** متشككش أبداً في توقيت خالد للنشر.

---

## Root Cause Analysis (عند الفشل فقط)

لو الحكم Failed 🔴، حدّد السبب الجذري من الفئات:
- خبر مفاجئ
- سوء قراءة فنية
- تجاهل إشارة زمنية
- سوء إدارة سيناريو

سجّله في `errors-log` مع **lesson_learned**.

---

## المخرجات

- **النجاح** → بوست "توقعنا اتحقق" (أقوى إثبات مصداقية للقناة).
- **الفشل** → feedback داخلي للـ Agent المسؤول. **مفيش بوست** عند الفشل.
