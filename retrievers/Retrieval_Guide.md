# دليل الاسترجاع | Retrieval Guide

**Category:** Retrievers
**الغرض:** المرجع الأساسي لأي Workflow أو n8n Retriever في اختيار ملفات المعرفة وقت التشغيل.

> المبدأ الحاكم: **لا تحقن كل المعرفة. احقن أقل ملفات تكفي للمهمة.** كل توكن زائد = تكلفة + تشويش تحليلي.

---

## 1. قاعدة الاسترجاع العامة

```
الأمر / نوع المهمة
      ↓
حدّد الـ Agent (من Switch)
      ↓
افتح بطاقة الـ Agent في agents/ → خد قائمة "المعرفة المطلوبة"
      ↓
احقن ملفات المعرفة دي فقط + البرومبت من prompts/
      ↓
نفّذ
```

## 3. متى تجمع بين أكثر من ملف

اجمع **فقط** لما المهمة تتطلب فعلاً:
- تحليل فني كامل = جان + (وايكوف/داو) + دخول + مخاطر. (4-6 ملفات كحد أقصى).
- لو السؤال عن نقطة واحدة (مثلاً "إيه الكسر الكاذب؟") = ملف واحد بس (`false-break-spring-upthrust.md`).
- لو الـ Writer بياخد تحليل جاهز = ملفات الأسلوب فقط، **مش** ملفات المنهج (التحليل اتعمل خلاص).

## 4. أولويات الاسترجاع (لما التوكنز محدودة)

| الأولوية | الملفات | السبب |
|----------|---------|-------|
| 1 (إلزامي) | البرومبت من `prompts/` | بدونه الـ Agent مش هيشتغل صح |
| 2 (إلزامي للتحليل) | جان + إدارة المخاطر | الأساس + الأمان |
| 3 (مهم) | الملف الأخص بالمهمة | السلوك السعري / الدخول |
| 4 (عند الحاجة) | فيبو / أشكال / مؤشرات | تفاصيل إضافية |
| 5 (نادر) | إليوت | الركن الثالث، موثّق جزئياً |

> لو لازم تقص: ابدأ بقص الأولوية 5 ثم 4. **متقصّش 1 و2 أبداً.**

## 5. قواعد تقليل التوكنز

1. **ملف واحد للسؤال الواحد** — السؤال المحدد ما يحتاجش كل المجال.
2. **مفيش تحميل استباقي** — ما تحملش ملف "احتياطي" مش متأكد إنك محتاجه.
3. **الكتابة ≠ التحليل** — الـ Writer ما بيحملش ملفات منهج.
4. **استخدم الفهرس أولاً** — `MarKeTia_KNOWLEDGE_INDEX.xlsx`: دوّر بالـ Keywords، خد المسار، حمّل ملف واحد.
5. **القراءة المركبة للسوق منفصلة** — ما بتتحملش مع التحليل الفردي.
6. **max_tokens ~600** للتحليل (كافي للبوست). متزودهاش بلا داعي.
7. **Related Topics مش أمر تحميل** — هي للسياق لو احتجت، مش لازم تحمّلها كلها.

---

## 6. ربط بالفهرس

الفهرس `MarKeTia_KNOWLEDGE_INDEX.xlsx` (عمود Source File) هو خريطة المسارات. الـ Retriever:
1. ياخد الـ keyword من نية المستخدم/الأمر.
2. يلاقي الصف في الفهرس.
3. يقرأ ملف الـ `Source File` ده فقط.
4. لو `Related Topics` لازمة للسياق، يحمّلها بالأولوية المذكورة فوق.

> 🔎 مستقبلاً (RAG / Vector DB): كل ملف معرفة chunk جاهز. الـ embeddings تتبني على مستوى الملف، والـ Retrieval_Guide ده بيتحول لقواعد re-ranking. التصميم الحالي جاهز للترقية بدون إعادة هيكلة.
