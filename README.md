# ماركيتيا | قاعدة المعرفة والمنظومة (v1.0)

> **المصدر الرسمي الوحيد (Single Source of Truth)** لكل الـ AI Agents في ماركيتيا.

## الهيكل المعماري (3 طبقات + دعم)

```
marketia_kb/
├── 00_vision/ .. 06_sop/   ← [Knowledge] المعرفة فقط (منهج/قواعد/أمثلة) — مفيش تعليمات تشغيل
├── prompts/                ← [System Prompts] البرومبتات التنفيذية المختصرة — مفيش شرح منهج
├── 05_agents/              ← بطاقات تعريف خفيفة للـ Agents (هوية + المعرفة المطلوبة + لينك البرومبت)
├── retrievers/             ← Retrieval_Guide.md: قواعد اختيار المعرفة وقت التشغيل
├── workflows/              ← توثيق تسلسل التنفيذ
└── configs/                ← الإعدادات التقنية العامة
```

## مبدأ الفصل (مهم)
- **المعرفة** (`00`–`06`): إيه المنهج والقواعد. ثابتة. مفيش تعليمات AI.
- **البرومبت** (`prompts/`): إزاي الـ Agent يفكر وينفّذ. **ما بيكررش** شرح المعرفة — بيشاور عليها.
- **الاسترجاع** (`retrievers/`): الـ n8n بيحمّل أقل ملفات تكفي المهمة (توفير توكنز).

## إزاي تستخدمه (Agent / Workflow)
1. ابدأ بـ `MarKeTia_KNOWLEDGE_INDEX.xlsx` (الفهرس) — دوّر بالـ Keywords.
2. حدّد الـ Agent → افتح بطاقته في `05_agents/` → خد البرومبت + المعرفة المطلوبة.
3. طبّق `retrievers/Retrieval_Guide.md` لتحميل أقل ملفات معرفة.
4. نفّذ.

## منهج التحليل (ترتيب صارم)
**الزمن (Gann) → السعر (Wyckoff + Price Action) → الموجي (Elliott)**

## ملاحظة عن البيانات
الجداول الحية (Predictions/Posts/Growth...) في Google Sheets. المعرفة التأسيسية هنا. الفصل مقصود.

جاهز لـ RAG / Vector DB بدون إعادة هيكلة. Agent Specifications الكاملة مؤجلة للإصدار القادم (مع RAG/Orchestrator أعقد).
